**Step 5 — Replace dispatch block inside _summarize_readme_with_llm (lines 258-275):**
Replace the entire try block's dispatch:
```python
if self.provider == "openai":
    response = openai_chat_completion(
        self.llm_client,
        model=self.model,
        messages=[{"role": "user", "content": prompt}],
        temperature=self.temperature,
        max_tokens=self.max_tokens,
    )
    return response.choices[0].message.content.strip()

elif self.provider == "anthropic":
    response = self.llm_client.messages.create(
        model=self.model,
        max_tokens=self.max_tokens,
        temperature=self.temperature,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text.strip()
```
With:
```python
response = llm_client.completion(
    model=llm_client.DEFAULT_MODEL,
    messages=[{"role": "user", "content": prompt}],
    temperature=self.temperature,
    max_tokens=self.max_tokens,
)
return response.choices[0].message.content.strip()
```

Note: The existing `except Exception` block and fallback to `_generate_structure_based_overview` on error stay — this is correct behavior to preserve.

Remove the unreachable `return self._generate_structure_based_overview(repo_name, file_structure)` line after the try/except (line 281) — it was only reachable if provider was neither "openai" nor "anthropic", which can no longer happen.
