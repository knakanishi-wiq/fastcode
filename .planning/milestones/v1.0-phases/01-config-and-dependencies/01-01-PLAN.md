<task type="auto">
  <name>Task 2: Create VertexAI smoke test</name>
  <files>tests/__init__.py, tests/test_vertexai_smoke.py</files>
  <action>
1. **Create tests/ directory** — Create `tests/__init__.py` (empty file) so pytest discovers the directory as a package.

2. **Create tests/test_vertexai_smoke.py** — Write a standalone pytest smoke test with two test cases in a `TestVertexAISmoke` class:

   **Happy path** (`test_happy_path_returns_valid_response`):
   - Load `.env` via `load_dotenv()` at module level
   - Skip with `@pytest.mark.skipif(not os.environ.get("VERTEXAI_PROJECT"), reason="VERTEXAI_PROJECT not set — skipping live test")` so CI without credentials passes cleanly
   - Call `litellm.completion(model="vertex_ai/gemini-3-flash-preview", messages=[{"role": "user", "content": "Say hello in one word."}])`
   - Assert `response.choices[0].message.content` is truthy (non-empty string)
   - Assert `response.model` is not None

   **Error path** (`test_missing_project_raises_config_error`):
   - Use `monkeypatch.delenv("VERTEXAI_PROJECT", raising=False)` and `monkeypatch.delenv("VERTEXAI_LOCATION", raising=False)` to remove env vars
   - Also `monkeypatch.delenv("GOOGLE_CLOUD_PROJECT", raising=False)` to prevent ADC fallback
   - Call `litellm.completion(...)` inside `pytest.raises(Exception)`
   - Assert the error text contains at least one of: "project", "credentials", "configuration", "not found", "default" (case-insensitive, broad keyword matching per research recommendation)
   - Assert "401" is NOT in the error text (the whole point: config error, not misleading auth error)
   - Do NOT assert exact error strings — litellm error messages change across versions

   **Constants at module level:**
   ```python
   VERTEX_MODEL = "vertex_ai/gemini-3-flash-preview"
   ```

   **Important:** Use `vertex_ai/gemini-3-flash-preview` as the model string (per user's locked decision). Do NOT use `gemini-2.0-flash-001` from the original roadmap. Do NOT use `gemini/` prefix (that routes to Google AI Studio, not VertexAI).

3. **Run the smoke test** — Execute `pytest tests/test_vertexai_smoke.py -v` to verify:
   - If `VERTEXAI_PROJECT` is set in `.env`: both tests should run (happy path passes, error path passes)
   - If `VERTEXAI_PROJECT` is NOT set: happy path is skipped, error path passes
