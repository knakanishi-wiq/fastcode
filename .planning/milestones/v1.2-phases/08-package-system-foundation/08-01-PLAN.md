**Critical rules:**
- The `[build-system]` block with hatchling is REQUIRED (PKG-01). Without it, uv treats this as a "virtual project" — fastcode is NOT installed as a package, only deps are resolved.
- pytest, pytest-asyncio, pytest-cov go ONLY in `[dependency-groups] dev` — never in `[project.dependencies]`.
- Use `[dependency-groups]` (PEP 735), NOT `[project.optional-dependencies]` for dev tools.
- Carry over the `litellm[google]>=1.80.8` lower-bound pin exactly as it appears in requirements.txt.
- Keep all other packages as bare names (no version pins) — let uv resolve to latest compatible.
- Do NOT add `[tool.hatch.build]` configuration unless the post-install verify step fails. Hatchling auto-discovers `fastcode/` at repo root.

<task type="auto">
  <name>Task 2: Generate uv.lock and verify editable install</name>
  <files>uv.lock</files>
  <action>
Run `uv lock` to generate the lockfile, then verify the editable install and git tracking.

Step-by-step:

1. Generate lockfile:
   ```bash
   uv lock
   ```
   Expected: resolves all deps and writes `uv.lock`. If resolution fails with conflicts, check uv output — add upper-bound pins only where uv explicitly reports incompatibility.

2. Verify uv.lock is NOT gitignored (critical — PKG-03 requires it be committed):
   ```bash
   git check-ignore -v uv.lock
   ```
   If this outputs a matching gitignore rule, add `!uv.lock` to `.gitignore` to un-ignore it.

3. Run `uv sync` to install into .venv:
   ```bash
   uv sync
   ```
   Expected: installs all runtime deps + dev group + fastcode editable.

4. Verify fastcode is importable as an installed package (PKG-01 key criterion):
   ```bash
   .venv/bin/python -c "import fastcode; print(fastcode.__file__)"
   ```
   Expected output: path ending in `fastcode/__init__.py` within the repo root (not a .venv site-packages copy).

   If this fails with ModuleNotFoundError and hatchling failed to auto-discover the package, add this to pyproject.toml and re-run `uv sync`:
   ```toml
   [tool.hatch.build.targets.wheel]
   packages = ["fastcode"]
   ```

5. Verify dev isolation (PKG-02):
   ```bash
   uv sync --no-dev
   .venv/bin/python -m pytest
   ```
   Expected: error like `No module named pytest` — NOT pytest running. This confirms pytest is correctly isolated in [dependency-groups] dev.

6. Stage the lockfile (do not commit yet — commit happens in Plan 02 after requirements.txt deletion):
   ```bash
   git add uv.lock pyproject.toml
   ```
   Verify tracking: `git ls-files uv.lock` must output `uv.lock`.
  </action>
  <verify>
    <automated>git ls-files uv.lock | grep -q "uv.lock" && echo "uv.lock tracked" || echo "FAIL: uv.lock not tracked by git"</automated>
    <manual>Check that `uv sync` completed without errors and `.venv/bin/python -c "import fastcode"` succeeds</manual>
  </verify>
  <done>uv.lock exists and is tracked by git (`git ls-files uv.lock` returns `uv.lock`); `uv sync` completes without errors; `import fastcode` succeeds in .venv; `uv sync --no-dev` followed by `.venv/bin/python -m pytest` produces ImportError (pytest absent)</done>
