<feature>
  <name>index_repo() — SQLite chunk writer with change detection</name>
  <files>fastcode/indexer.py, tests/test_indexer_sqlite.py</files>

  <behavior>
    index_repo(repo_path: str, db_path: str, max_file_size_bytes: int = 1_000_000) -> dict

    CHANGE DETECTION (IDX-02):
    - For each file discovered: check stored mtime_ns in sources table
      - If mtime_ns matches exactly: skip (no hash, no parse) → increment stats["skipped"]
      - If mtime_ns differs or file is new: compute SHA-256(file bytes), compare to stored content_hash
        - If hash matches: update mtime_ns only → increment stats["skipped"]
        - If hash differs or file is new: delete sources row (cascade deletes chunks), reinsert → increment stats["indexed"]
    - After walking filesystem: delete sources rows for paths no longer on disk → increment stats["deleted"] per deleted path

    CHUNK WRITING (IDX-01):
    - For each changed/new file: call CodeParser({}).parse_file(file_path, content) to get FileParseResult
    - Extract chunks from the parse result: each function and class is one chunk. If neither classes nor
      functions exist (e.g. small utility file), treat the entire file content as one chunk.
    - For each chunk: extract content text from file lines using start_line/end_line (1-indexed, inclusive)
    - Insert into chunks: (source_path=relative_path, content=chunk_text, content_hash=sha256(chunk_text.encode()),
      chunk_index=N, start_offset=start_line, end_offset=end_line)
    - Insert into sources: (path=relative_path, content_hash=sha256(file_bytes), mtime_ns=stat.st_mtime_ns, size=stat.st_size)
    - Transaction scope: one transaction per file (commit after sources + chunks inserts for that file)

    FILE FILTERING:
    - Use pathspec.PathSpec.from_lines("gitwildmatch", patterns) to parse repo's .gitignore
    - Skip any path matched by .gitignore patterns
    - Skip hidden directories (dir name starts with ".")
    - Skip files larger than max_file_size_bytes (default 1_000_000)
    - Delegate extension filtering to CodeParser: call parse_file() and treat None result as unsupported extension → skip silently
    - Walk tree with os.walk(repo_path); prune hidden dirs from dirs list in-place

    ERROR HANDLING:
    - On Exception during a file's processing: log WARNING with file path and error message; increment stats["errors"]; continue
    - index_repo() calls init_db(db_path) at the start before processing any files

    RETURN VALUE:
    - {"indexed": int, "skipped": int, "deleted": int, "errors": int}

    LOGGING:
    - logging.getLogger(__name__) at module level
    - INFO per file action: "Indexed {path}", "Skipped {path} (unchanged)", "Deleted {path} (removed from disk)"
    - WARNING for errors: "Failed to index {path}: {error}"

    Test cases (input → expected output):
    - Empty dir → {indexed: 0, skipped: 0, deleted: 0, errors: 0}
    - Dir with one Python file → {indexed: 1, skipped: 0, deleted: 0, errors: 0}, chunks table non-empty
    - Re-index same dir unchanged → {indexed: 0, skipped: 1, deleted: 0, errors: 0}, chunk count unchanged
    - Re-index after deleting a file → {indexed: 0, skipped: 0, deleted: 1, errors: 0}, chunks removed
    - Re-index after modifying a file's content → {indexed: 1, skipped: 0, deleted: 0, errors: 0}, new chunks
    - File in .gitignore → not indexed, not in chunks/sources
    - Hidden dir (e.g. .git/) → contents not indexed
    - File larger than max_file_size_bytes → skipped silently (not counted in skipped, not in indexed)
    - Unsupported extension (e.g. .txt) → parse_file returns None → silently skipped
  </behavior>

  <implementation>
    Add index_repo() as a module-level function at the BOTTOM of fastcode/indexer.py (after the
    CodeIndexer class). This avoids disrupting the existing class and keeps the new function
    co-located with the existing indexing code.

    Required imports to add at the top of fastcode/indexer.py (only add if not already present):
      import os
      import stat as stat_mod  (only if needed for stat constants; os.stat() is fine directly)
      import sqlite3
      import pathspec
      from .db import init_db

    The function reads .gitignore from repo_path/.gitignore using pathspec.PathSpec.from_lines().
    If no .gitignore exists, use an empty spec (matches nothing).

    Use os.walk(repo_path, topdown=True) and prune hidden dirs:
      dirs[:] = [d for d in dirs if not d.startswith('.')]

    For computing relative paths inside the repo: use os.path.relpath(file_path, repo_path).

    For SHA-256: hashlib.sha256(file_bytes).hexdigest() where file_bytes = open(path, 'rb').read()

    For chunk extraction from FileParseResult:
      chunks = []
      lines = content.split('\n')  # 0-indexed list
      for i, cls in enumerate(parse_result.classes):
          chunk_lines = lines[cls.start_line - 1 : cls.end_line]
          chunks.append((cls.start_line, cls.end_line, '\n'.join(chunk_lines)))
      for i, func in enumerate(parse_result.functions):
          chunk_lines = lines[func.start_line - 1 : func.end_line]
          chunks.append((func.start_line, func.end_line, '\n'.join(chunk_lines)))
      if not chunks:
          chunks = [(1, len(lines), content)]

    Deduplicate chunks by start_line to avoid double-counting methods already covered by class chunks:
    use dict keyed on start_line and take first occurrence.

    For each chunk in deduplicated list, assign chunk_index = enumerate order (0-based).
    Compute content_hash = hashlib.sha256(chunk_text.encode('utf-8', errors='replace')).hexdigest()

    Transaction pattern per file:
      with conn:  # auto-commits on exit, rolls back on exception
          conn.execute("DELETE FROM sources WHERE path = ?", (rel_path,))
          conn.execute("INSERT INTO sources ...", (...))
          for chunk_index, (start, end, text) in enumerate(chunks):
              conn.execute("INSERT INTO chunks ...", (...))

    (The DELETE FROM sources cascades to chunks automatically.)
  </implementation>
</feature>
